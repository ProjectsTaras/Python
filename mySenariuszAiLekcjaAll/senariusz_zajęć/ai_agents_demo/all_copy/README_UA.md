# Демонстрація multi-agent для уроку

Цей мініпроєкт показує учням, що `AI-агенти` це не просто "ще один чатбот", а система, де кілька окремих ролей співпрацюють між собою.

## Що є в проєкті

- `multi_agent_demo.py` - готовий скрипт для показу на уроці
- `multi_agent_demo.ipynb` - та сама демонстрація у форматі Jupyter Notebook
- `app.py` - локальний вебсервер для візуальної демонстрації в браузері
- `web/` - інтерфейс, де видно агентів, кроки і передачу результатів

## Що саме демонструється

Ситуація:

`Школа хоче вирішити, чи варто впроваджувати AI-асистента.`

У програмі працюють 5 агентів:

1. `PlannerAgent` - ділить завдання на етапи
2. `BenefitsAgent` - збирає аргументи "за"
3. `RisksAgent` - збирає ризики
4. `SafetyAgent` - пропонує правила безпечного використання
5. `DecisionAgent` - об'єднує попередні результати і формує рекомендацію

## Як запустити

У терміналі:

```powershell
python .\multi_agent_demo.py
```

Для візуальної версії в браузері:

```powershell
python .\app.py
```

Після цього відкрий у браузері:

`http://127.0.0.1:8000`

Або відкрий:

`multi_agent_demo.ipynb`

## Як пояснювати учням

Головна думка уроку:

`Один чатбот може дати відповідь одразу, але agent system ділить складну задачу на ролі, перевіряє її з різних сторін і поетапно формує рішення.`

## Що можна сказати під час показу

- "Зараз ви побачите не один AI, а кілька агентів з різними функціями."
- "Кожен агент отримує свою частину роботи."
- "Фінальна відповідь створюється після передачі результатів між агентами."
- "Саме в цьому і полягає ідея agentic systems."

## Важливо чесно пояснити

Ця версія є `навчальною симуляцією архітектури агентів`. Вона справді показує:
- поділ задачі на ролі;
- порядок виконання;
- залежність одного агента від результатів інших;
- оркестрацію багатокрокового процесу.

Але це ще не `живі LLM-агенти` з API. Якщо захочеш, я можу наступним кроком додати версію з реальною моделлю через API, щоб агенти не мали заздалегідь записаних відповідей, а генерували їх під час запуску.
